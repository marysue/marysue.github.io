IT Application : Watcher

PowerShell:
In this event driven script I make files available to Dreamweaver in our virtualized environment. Dreamweaver expects files to be located in c:/inetpub/www directory. However, due to the virtualization process we destroy the VM when the user logs out - which means the c: drive is destroyed. So to handle this, we store the user's files on a network mounted drive. Why not repoint Dreamweaver? Well, Dreamweaver was never intended to be "repointed" to a different storage area. To appease Dreamweaver when the user starts the program the user files are copied from their network storage to c:/inetpub/www directory. When a file is touched (modified, added, or deleted), this is mirrored on the user's network mounted drive from the local c:/inetpub/www directory. to preserve the files before the virtual machine is destroyed.
